模块按 FastAPI 标准三层划分：
- `routers/auth.py` 定义 `/api/auth` 路由，仅做参数绑定、依赖注入（`Depends(get_db)`、`Depends(get_current_user)`）和异常转换，不承载业务逻辑；通过 `_build_token_response` 统一构造 `TokenResponse`。
- `services/auth.py` 封装所有业务操作（注册、认证、改密、软删除等），以 `AsyncSession` + SQLAlchemy 2.0 `select()` 风格访问 `User` 模型，并通过 `app.utils.security` 提供的 `create_access_token` / `hash_password` / `verify_password` 完成 JWT 签发与密码处理；错误一律抛出 `ValueError` 由路由层捕获并转为 `HTTPException`。
- `schemas/auth.py` 使用 Pydantic v2（`field_validator`、`model_config.from_attributes`）集中声明请求/响应体，对邮箱、用户名、性别、密码强度等进行校验。
依赖方向严格单向：router → service → (models/schemas/utils.security)，avatar 上传在 router 层直接落盘到 `uploads/avatars/`，旧文件在写入前清理。

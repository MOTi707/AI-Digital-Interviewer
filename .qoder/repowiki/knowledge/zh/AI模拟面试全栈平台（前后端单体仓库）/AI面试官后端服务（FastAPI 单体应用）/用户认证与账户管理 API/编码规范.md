- 路由层将业务异常包装为 HTTPException（400/401），服务层只抛 ValueError，保持错误传播一致。
- 数据库读写统一采用 SQLAlchemy 2.0 异步 `select()` + `db.execute()` + `scalar_one_or_none()` 模式。
- Pydantic 校验集中在 schema 的 `@field_validator` 中，并对空字符串进行归一化（如 gender 转 None）。
- 返回给客户端的 ORM 对象一律通过 `UserResponse.model_validate(...)` 序列化，避免直接暴露模型字段。
- 需要刷新 ID 或关联对象的写操作遵循 `db.flush()` + `db.refresh()` 两步提交后再返回。

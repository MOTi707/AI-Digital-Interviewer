模块按 FastAPI 标准三层划分：
- `routers/post.py` 定义 `/api/posts` 路由组（tags="面经论坛"），仅做参数解析、鉴权委派与响应组装；通过 `_optional_user` 依赖实现可选认证（无 token 时以匿名身份访问）。
- `services/post_service.py` 承载所有业务与数据访问：使用 `sqlalchemy` 的 `select`/`and_`/`or_` 构建动态条件，支持公司、岗位、年份、状态、面试类型、关键词以及多标签精确匹配（通过 `post_tags` 中间表 + `group_by`+`having count == len(tag_names)` 实现交集过滤），并提供 `_build_post_response` 将 ORM 对象转为 Pydantic Response。
- `schemas/post.py` 集中声明请求/响应 Pydantic v2 模型（`PostCreate`、`PostResponse`、`CommentCreate`、`TagStatResponse` 等），统一设置 `model_config = {"from_attributes": True}` 以直接映射 ORM 字段。
依赖方向单向：router → service → models/schemas；router 还调用同层 `comment_service` 处理评论子资源，并通过 `_get_base_url()` 硬编码前端地址生成分享链接。

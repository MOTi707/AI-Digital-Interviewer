- 所有管理端路由统一以 `Depends(_require_admin)` 作为第一个依赖参数进行权限拦截，失败时抛出 403。
- 分页接口遵循 keyword/page/size 三参约定，并在路由层用 `Query(..., ge=1, le=100)` 约束范围。
- service 层的更新方法接收 `dict` 并通过 `for key, value in data.items(): if value is not None: setattr(obj, key, value)` 做增量更新，删除方法返回布尔值表示是否找到记录。
- 列表查询先对原始 `select` 构造子查询执行 `func.count()` 得到 total，再在原查询上追加 `order_by(created_at.desc()).offset().limit()` 完成分页。
- 路由层将 service 返回的 ORM 对象统一通过对应 `AdminXxxItem.model_validate(...)` 转为 Pydantic response_model，保证对外字段一致。

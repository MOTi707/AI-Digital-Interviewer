- 路由函数统一通过 `Depends(get_db)` 和 `Depends(get_current_user)` 注入数据库会话与当前用户，并在缺少资源时抛出 `HTTPException(status_code=404, detail=...)`。
- 对外部 AI 能力（结构化解析、措辞优化）均以 `resume_service.has_api_key()` 前置检查，未配置时返回 400 并提示在 `.env` 中设置 `DEEPSEEK_API_KEY`。
- 服务层对同一用户的简历采用 UPSERT 模式：先 `get_resume_by_user` 判断存在性，存在则就地更新字段并 `flush+refresh`，不存在则新建后 `add+flush+refresh`。
- Pydantic 响应模型使用 `model_config = {"from_attributes": True}` 直接由 ORM 实例构造，避免手动字段映射。
- Deepseek 调用统一走 `_api_url/_api_key/_model` 三个小助手从 `settings` 取值，并通过正则 `r"```(?:json)?\s*([\s\S]*?)```"` 剥离 markdown code block 后再 `json.loads`。

- 路由函数统一以 `Depends(get_current_user)` 注入用户、`Depends(get_db)` 注入 `AsyncSession`，并在进入业务前对 session 进行存在性与 `status == 'in_progress'` 双重校验，不存在则抛 404、已结束则抛 400。
- 响应体一律通过显式构造 Pydantic Response 类字段的方式组装（而非直接返回 ORM 对象），并复用 `interview_service._build_rounds_progress` 计算 `rounds_progress`。
- 数据库查询使用 `select(...).where(...)` 搭配 `db.execute(...).scalar_one_or_none()` / `scalars().all()` 模式，避免原生 SQL 字符串。
- 对外部 LLM 调用统一封装为 try/except 块，异常时回退到默认分数（如 8.0）并附带错误提示，保证服务可用性。
- 轮次相关常量集中定义于模块级（`ROUNDS`、`ROUND_KEYS`、`AI_INTERVIEW_PROMPTS`），通过 key 值驱动分支逻辑，新增轮次只需扩展常量表。

本模块采用三层划分：
- `routers/interview.py`：FastAPI `APIRouter(prefix='/api/interview')`，仅做参数校验、鉴权（`Depends(get_current_user)`）、会话状态前置检查（`in_progress`）和响应体拼装，所有业务逻辑委托给 service。
- `services/interview_service.py`：核心领域层，维护 ROUNDS 顺序、JOB_CATEGORIES 与 ASSESSMENT_SEED/BUSINESS_SEED 等静态种子数据；实现会话 CRUD、按轮次选题（assessment 随机10题、tech 从 OJ 抽题、business 按岗位过滤、ai_voice_* 返回首问）、答题评分（选择题/判断题直接比对、tech 复用 `problem_service.create_submission`、AI 面通过 `_grade_ai_answer` 调用 DeepSeek）、SSE 流式对话（`httpx.AsyncClient.stream` + `data:` 行解析）、轮次推进、切屏上报与综合报告生成（含雷达图维度映射与等级判定）。
- `schemas/interview.py`：纯 Pydantic v2 请求/响应模型，使用 `model_config = {'from_attributes': True}` 将 SQLAlchemy 模型直接转为响应。
依赖方向单向：router → service → (models.interview / models.problem / services.problem_service)；service 通过 `app.config.get_settings()` 读取 DeepSeek API 配置，外部 HTTP 客户端统一用 httpx AsyncClient。

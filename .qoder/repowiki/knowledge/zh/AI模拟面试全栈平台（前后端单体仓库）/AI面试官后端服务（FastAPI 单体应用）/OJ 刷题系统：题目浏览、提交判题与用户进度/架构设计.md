本模块遵循典型的三层结构：
- `routers/problem.py`：FastAPI 路由层，定义 `/api/problems` 前缀下的 REST 端点（列表、标签选项、详情、提交、调试、进度），通过 `_optional_user` 依赖实现可选认证（无 token 时匿名访问），所有业务调用委托给 `problem_service`。
- `services/problem_service.py`：领域服务层，封装 SQLAlchemy 异步查询（`get_problems`、`check_user_solved`、`get_user_progress` 等）、提交判题流程（解析样例 JSON → 调用 `app.services.code_executor.execute_code` → 逐组比对输出 → 持久化 `Submission` 并更新 `Problem` 统计）、调试执行以及 `build_problem_list_item` / `build_problem_detail` 等 ORM→Pydantic 转换辅助函数；底部附带 `SEED_PROBLEMS` 常量与 `init_seed_problems` 用于首次启动填充示例题库。
- `schemas/problem.py`：Pydantic v2 请求/响应模型，集中声明支持的语言白名单 `VALID_LANGUAGES`，并通过 `field_validator` 校验语言字段。

依赖方向单向向下：router → service → (models + code_executor + schemas)，router 不直接操作数据库或外部执行器。

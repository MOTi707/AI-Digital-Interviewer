- store 方法统一返回 `{ success: boolean; message: string }` 结果对象，视图侧根据 `success` 分支显示成功/错误消息，而非抛出异常。
- 所有 API 请求通过 `apiRequest<T>(path, options)` 统一封装，自动从 `localStorage.getItem('auth_token')` 注入 `Authorization: Bearer` 头，并在非 2xx 时解析 `body.detail` 作为错误信息。
- 会话数据同时写入内存 ref（`user`/`token`）与 localStorage（`auth_token`、`auth_user`），登录/注册/更新后由 `_setSession` 或 `updateProfile` 同步两端。
- 表单交互采用 `ref` 控制 loading 状态（如 `isSubmitting`、`profileSaving`、`usernameLoading`），提交前清空 `errorMessage`/`successMessage`，finally 中重置 loading。
- UI 风格遵循 Memphis Neo-brutalism：按钮/卡片统一使用 `border-4 border-black shadow-[Xpx_Ypx_0px_0px_rgba(0,0,0,1)] hover:shadow-none hover:translate-x-[N] hover:translate-y-[N]` 组合，颜色来自 `bg-memphis-*` 语义类。

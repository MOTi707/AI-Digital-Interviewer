模块由两个 Vue 单文件组件 + 一个 Pinia Store 组成：
- `ResumeView.vue`（/resume）：主入口，按 `resumeStore.hasResume / hasParsedContent` 三态渲染——无简历上传区、双栏分析页（左侧结构化提取 skills/experiences/education，右侧综合评分+技能词云+经历时间线+技能分布+改进建议），并内嵌上传弹窗与原文查看弹窗。
- `ResumeOptimizeView.vue`（/resume/optimize）：通过路由从主页面进入，采用左右 diff 布局展示原文删除行与优化新增行，底部统计卡片汇总优化条数、专业度提升等指标；调用 store 的流式接口边接收边追加 DOM。
- `stores/resume.ts`：唯一状态源，集中封装所有 `/api/resume/*` 请求（`fetchConfig`、`fetchResume`、`uploadResume`、`analyzeResume`、`optimizeText`、`extractPdfText`），并提供 `optimizeTextStream` 用 `ReadableStream` + `TextDecoder` 手动解析后端 SSE（`data: {type:'start'|'item'|'done'}`）。PDF 文本优先走后端 `/api/resume/extract-text`（PyMuPDF），DOCX 则动态 import `mammoth` 在前端解析。
依赖方向：两个 View 仅依赖 `useResumeStore`，不直接访问 API；鉴权 token 统一从 `localStorage.auth_token` 注入到 `Authorization: Bearer ...` 头。

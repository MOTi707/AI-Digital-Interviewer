入口 `src/main.ts` 创建 Vue 应用并挂载 Pinia、Router，随后调用 `useAuthStore().init()` 恢复本地 token，完成后才 mount App，保证所有子模块在启动时即可读取会话状态。

- 路由集中声明于 `src/router/index.ts`，通过 `meta.requiresAuth` / `meta.requiresAdmin` 配合全局 `beforeEach` 守卫实现用户态与管理端态的访问控制；布局由 `meta.layout` 驱动，App.vue 根据 layout 渲染 `DefaultLayout` / `DashboardLayout` / `AdminLayout`。
- 状态按业务域拆分为独立 Pinia store（`auth.ts`、`resume.ts`、`interview.ts`、`career.ts`、`oj.ts`、`forum.ts`、`admin.ts`），各子模块（views + components + store）围绕同一份 store 协作，避免跨模块直接 import。
- 静态资源（VRM 模型、MBTI SVG、Logo）统一放在 `public/`，由后端 SSE/流式接口驱动的 TTS 能力集中在 `utils/tts.ts`，被面试与简历优化等模块复用。

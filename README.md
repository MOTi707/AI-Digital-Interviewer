# AI面试官 - 智能求职准备平台

> 一站式 AI 驱动的求职准备系统，涵盖模拟面试、在线刷题、职业测评、简历优化与面经社区，助力求职者全方位提升面试能力。

---

## 项目简介

**AI面试官**是一个面向求职者的全栈 Web 应用，通过集成大语言模型（DeepSeek）、语音合成（Edge-TTS）与语音识别（讯飞语音听写）等 AI 能力，为用户提供高度仿真的面试训练体验。系统支持多轮面试流程模拟，从综合素质测评到技术面、业务面再到 AI 语音面试，覆盖了真实求职面试的完整链路。同时内置在线编程练习（OJ）、职业性格测评（MBTI / 霍兰德）、AI 简历优化以及面经分享社区，构建了一个功能闭环的求职准备生态。

## 核心功能

### 一、AI 模拟面试

模拟面试是本系统的核心功能，完整复刻了企业面试的多轮流程。

**面试轮次设计：**

| 轮次 | 名称 | 说明 |
|------|------|------|
| 第一轮 | 综合素质测评 | 10 道选择题，考察职业价值观、沟通能力、抗压能力等 |
| 第二轮 | 一面·技术面 | 技术判断题与编程题，考察专业知识深度 |
| 第三轮 | 二面·业务面 | 开放式问答题，考察业务理解与项目经验 |
| 第四轮 | 三面·AI 面试 | 语音交互式面试，集成 3D 虚拟面试官形象 |
| 第五轮 | 四面·综合面试 | 综合考察，语音交互深度问答 |

**功能亮点：**

- **12 个岗位方向**：覆盖互联网（Java 后端、前端、AI 工程师、测试、DevOps、产品经理）和公务员（综合管理、财政审计、法规执法等）两大行业类别
- **两种面试模式**：全流程模式（依次完成所有轮次）和单轮练习模式（针对性训练某一轮次）
- **3D 虚拟面试官**：基于 Three.js + VRM 模型渲染的 3D 虚拟人物，配合语音合成（TTS）实现语音提问，打造沉浸式面试体验
- **语音交互**：支持语音录入（ASR 语音识别）和语音播报（TTS 语音合成），用户可像真实面试一样进行口头回答
- **AI 智能评分**：每道题的回答由 DeepSeek 大模型进行多维度评分，并给出改进建议
- **防作弊机制**：记录切屏次数、禁用右键和开发者工具、全屏保护模式，还原真实面试的严肃环境
- **AI 多轮对话**：支持在面试过程中与 AI 面试官进行追问和深度交流（SSE 流式输出）
- **评分报告**：面试结束后生成多维度雷达图评分报告（专业能力、逻辑思维、沟通表达、岗位匹配度），包含每轮详细点评和综合改进建议

#### 3D 数字人面试官

数字人面试官是本系统最具技术含量的核心模块，基于 WebGL 实时渲染技术构建了一套完整的虚拟面试官系统。不同于传统面试软件的文字交互，本系统通过 3D 虚拟人物、语音合成、语音识别和 AI 对话四大能力的深度融合，实现了接近真人面对面的沉浸式面试体验。

#### 3D 渲染与模型系统

系统使用 **Three.js** 作为底层 WebGL 渲染引擎，配合 **@pixiv/three-vrm** 插件加载标准 VRM（Virtual Reality Model）格式的虚拟人物模型。VRM 是一种基于 glTF 2.0 的扩展格式，专为虚拟人物设计，包含完整的人体骨骼、表情系统和视线追踪数据。

**渲染管线配置：**
- 启用抗锯齿（antialias）和 PCF 软阴影映射，提升画面质量
- ACES Filmic 色调映射 + sRGB 色彩空间，色彩还原自然
- 四光源布局：环境光（0.8 强度）+ 主方向光（1.5 强度，投射阴影）+ 填充光（0.5）+ 轮廓光（0.3），模拟专业影棚布光效果
- 相机 FOV 30°，自动跟随容器尺寸自适应（ResizeObserver），支持设备像素比适配高 DPI 屏幕

**模型加载流程：**
1. 通过 GLTFLoader 解析 VRM 文件，注册 VRMLoaderPlugin 启用 VRM 扩展属性
2. VRMUtils 自动移除冗余关节和顶点，优化模型体量
3. 根据包围盒自动计算缩放比例，将模型归一化到标准尺寸
4. 居中定位模型，消除模型坐标系偏移
5. T-Pose 矫正：根据模型类型自动调整手臂骨骼旋转角度，恢复自然站姿
6. 绑定 LookAt 目标到相机对象，实现视线跟随

**多模型支持：** 内置两套 VRM 模型（通用面试官 / Eku 风格），用户可在面试过程中一键切换。切换时自动执行完整的资源清理流程（旧模型几何体、材质、纹理 GPU 内存释放）并重新走加载流程。

#### 表情系统

基于 VRM 标准的 Expression（BlendShape）系统，实现了六种情绪表情的平滑过渡和自然混合：

| 情绪 | 用途 | 触发时机 |
|------|------|----------|
| neutral（平静） | 默认状态，面试官保持专业从容 | 始终作为基础表情 |
| happy（微笑） | 候选人回答出色时给予正面反馈 | 用户发送回答后 |
| angry（严肃） | 追问环节表现认真和审视态度 | AI 追问时 |
| sad（遗憾） | 对不理想回答表达惋惜 | 特定评分触发 |
| relaxed（放松） | 思考时呈现沉思姿态 | AI 生成回复时 |
| surprised（惊讶） | 对意外回答表示关注 | AI 回复完成时 |

**平滑过渡算法：** 每种情绪维护当前权重和目标权重两组参数，以 `speed = 3.0` 的插值速度在每帧进行线性过渡（lerp），避免表情突变带来的机械感。情绪触发后默认 2 秒自动回归 neutral 状态，模拟真实面试官的表情自然变化。

**口型冲突处理：** 在面试官说话（TTS 播报）时，情绪表情权重自动降低至 30%（`expressionScale = 0.3`），优先保证口型动画的准确性，避免表情和口型抢夺 BlendShape 通道。

#### 视线追踪系统

三种视线模式无缝切换，让面试官的注视方向始终自然且符合交互语境：

- **Camera 模式**（默认）：注视摄像头位置，产生「直视用户」的效果，营造面对面交流的临场感
- **Mouse 模式**：鼠标在窗口内移动时自动激活，视线跟随鼠标位置映射到 3D 空间（左右 ±2.0，上下 ±1.2 的范围映射），模拟面试官环顾的自然感
- **Input 模式**：用户在输入框聚焦时触发，视线看向输入区域方向（3 秒后自动回退），暗示面试官在等待你的回答

**平滑过渡：** 目标视线位置使用 `lerp(target, delta * 5)` 逐帧插值，避免视线跳变。通过创建辅助 Object3D 作为 VRM LookAt 的代理目标点，间接控制眼球朝向，兼容 VRM 标准的视线系统。

#### 口型动画（Lip Sync）

面试官说话时，口型动画与 TTS 语音播放实时同步，实现「嘴型跟声音走」的自然效果：

- **五种口型循环**：aa（张嘴）、ih（扁嘴）、ou（圆唇）、ee（展唇）、oh（小圆唇），随机切换模拟自然说话节奏
- **随机间隔**：口型切换间隔在 0.08 ~ 0.20 秒之间随机变化，避免机械重复
- **随机幅度**：口型权重在 0.3 ~ 0.8 之间随机，模拟说话时的力度变化
- **自动停止**：TTS 播放结束时所有口型 BlendShape 立即归零，恢复闭嘴状态

#### 自然动作系统

基于 VRM Humanoid 骨骼系统，为面试官设计了四种行为状态，每种状态独立控制头部（Head）、脊柱（Spine）、双臂（UpperArm / LowerArm）共 6 个骨骼节点的旋转动画：

**Idle（待机）** — 面试开始前或等待用户输入时
- 头部三轴微小正弦摆动（频率 0.4~0.6 Hz），模拟自然的呼吸和微动
- 脊柱以更低频率（0.5~0.8 Hz）轻微晃动
- 双臂保持自然下垂姿态，带微弱摆动

**Speaking（说话）** — TTS 语音播报时
- 头部点头幅度加大（0.04 振幅 × 2.5 Hz），配合语句节奏
- 右手明显比划动作（前臂 -0.2 ± 0.15 rad，频率 2.5 Hz），模拟面试官讲解时的手势
- 左手保持自然，形成不对称的自然体态

**Thinking（思考）** — AI 正在生成回复时
- 头部微低、微侧（x: 0.05, z: 0.05），呈现沉思姿态
- **右手抬起摸下巴**（上臂 z=0.8, x=-0.6，前臂 x=-1.2），经典的思考动作
- 脊柱微前倾，整体传达「正在认真考虑你的回答」的信号

**Satisfied（满意）** — 用户回答后
- **明显点头动作**（头部 x 方向 0.08 振幅 × 3.0 Hz），传达认可
- 配合 happy 表情，形成「满意点头微笑」的复合反馈
- 双手保持自然放松

**Probing（追问）** — AI 回复完成后
- **身体明显前倾**（脊柱 x: 0.06），传达「我要深入问你」的压迫感
- 双手交叉或收敛（前臂 x: -0.5），严肃姿态
- 配合微 angry 表情（15% 权重），营造真实面试中追问环节的紧张氛围

#### 眨眼动画

- **随机间隔**：每 2~6 秒随机触发一次眨眼，避免规律性的机械感
- **正弦曲线驱动**：眨眼过程使用 `sin(progress * π)` 曲线，实现自然的「快合慢开」效果
- **与口型协调**：说话和安静状态下均正常工作，不产生动画冲突

#### TTS 语音合成（句子级流水线）

面试官的语音输出采用**句子级流式 TTS 流水线**，实现「AI 边生成文字，面试官边朗读」的极低延迟体验：

```
AI 流式输出 ──→ 句子切分 ──→ TTS 预取 ──→ 链式播放
     │              │             │            │
  SSE 逐字到达   按句号拆分    立即请求音频   播完自动接下一句
```

**核心机制：**

1. **句子级切分**：AI 回复的 SSE 流每到达一个句子终止符（`。！？；\n`），立即将该句送入 TTS 队列。智能处理括号未闭合的跨句画外音（如「（微笑点头）」），等待括号闭合后再切分
2. **音频预取**：句子入队后立即发起 Edge-TTS 后端请求，不等播放到该句才开始下载。当播放引擎播完当前句时，下一句的音频通常已经下载完毕
3. **链式播放引擎**：递归式的 playNext 机制，当前句播放结束自动取出队列中下一句，实现无缝衔接
4. **双引擎降级**：优先使用后端 Edge-TTS（高质量微软语音，流式 MP3 传输），若请求失败自动降级到浏览器内置 Web Speech API（优先选择 Xiaoxiao、Yaoyao 等中文音色）
5. **画外音过滤**：AI 回复中的动作描述（如「（微笑）」「（点头）」）在送入 TTS 前自动剥离，避免朗读出这些非对话文字

#### ASR 语音识别

用户可通过麦克风进行语音回答，系统将音频转换为文字后提交给 AI 面试官：

- **前端录音**：使用 MediaRecorder API 录制 WebM 格式音频（Opus 编码优先）
- **音频格式转换**：前端将 WebM 转为标准 WAV 格式（16kHz 单声道 16-bit PCM），构造完整 WAV 文件头（RIFF + WAVE + fmt + data），确保后端识别引擎兼容
- **后端识别**：通过讯飞语音听写 WebSocket API 进行中文语音识别，支持普通话非标发音适配
- **全链路控制**：录音开始时自动停止 TTS 播放（避免回声），识别完成后文字自动填入输入框供用户确认和编辑

#### 数字人面试的完整交互流程

一次数字人面试轮次（三面 / 四面）的完整交互链路如下：

```
用户进入面试 → 加载 VRM 模型 → AI 发出第一个问题（SSE 流式）
     ↓
句子到达 → TTS 预取音频 → 面试官朗读 + 口型动画 + 说话肢体动作
     ↓
朗读完成 → 切换为 Idle 状态 → 等待用户回答
     ↓
用户录音/输入 → 提交回答 → 面试官满意点头 + 微笑表情
     ↓
AI 生成追问（SSE） → 面试官思考动作（摸下巴）→ 逐句朗读
     ↓
面试官追问完毕 → 身体前倾 + 严肃表情 → 用户继续回答
     ↓
重复 6 轮对话 → 面试完成 → 生成评分报告
```

**异常处理与中止机制：**
- 用户主动中止面试时，立即通过 AbortController 取消在途的 SSE 请求、停止 TTS 队列、终止录音
- 切屏检测触发自动中止时，执行相同的清理流程
- 模型加载失败时显示友好提示，不影响面试流程继续进行（可仅使用文字交互）

### 二、在线编程练习（OJ 刷题系统）

内置在线编程判题系统，用户可在线编写、调试和提交代码。

**支持语言：**
- Python 3
- C
- C++
- Java
- JavaScript（Node.js）

**功能亮点：**

- **真实代码执行**：通过沙箱子进程实际编译和运行用户代码，而非模拟判定
- **安全沙箱**：多层安全策略，包含危险关键词黑名单（覆盖文件系统操作、进程注入、网络访问、系统命令等），按语言分别配置安全规则
- **执行反馈**：实时返回执行时间、内存消耗、标准输出和错误信息
- **调试模式**：支持自定义输入数据进行代码调试，方便排查逻辑错误
- **难度分级**：题目按 Easy / Medium / Hard 三级难度分类，支持标签筛选和关键词搜索
- **进度追踪**：记录用户的解题历史和通过状态，可视化刷题进度统计
- **内置题库**：系统启动时自动初始化种子题目，开箱即用

### 三、职业测评

内置两套专业职业性格测评量表，帮助用户认知自我、明确职业方向。

**测评类型：**

- **MBTI 性格测评**：基于 Myers-Briggs 类型指标，通过多维度问题评估用户的性格类型（16 种人格），推荐匹配的职业方向
- **霍兰德职业兴趣测评**：基于 Holland 六角模型，评估用户在现实型、研究型、艺术型、社会型、企业型、常规型六个维度的倾向

**功能亮点：**

- **Likert 五点量表**：专业标准化评分体系，确保测评结果的科学性和可靠性
- **AI 岗位推荐**：结合测评结果和用户简历技能关键词，由 DeepSeek 大模型生成个性化岗位推荐报告（SSE 流式输出），包含匹配岗位、推荐理由和备考建议
- **测评历史**：完整保存历次测评结果，支持随时回看和对比
- **可视化结果**：配合 ECharts 图表展示测评维度得分，直观呈现性格画像

### 四、AI 简历优化

基于 AI 的简历分析与措辞优化工具，帮助用户提升简历质量。

**功能亮点：**

- **多格式支持**：支持 PDF 和 DOCX 格式简历上传，服务端使用 PyMuPDF 精准提取 PDF 文本
- **AI 结构化分析**：调用 DeepSeek 大模型自动从简历原文中提取教育背景、工作经历、项目经验、技能特长等结构化信息
- **措辞优化**：AI 逐条分析简历描述，提供 Before/After 对照的措辞优化建议，量化改进统计（优化条数、提升幅度等）
- **SSE 流式输出**：优化结果逐条流式推送，边生成边展示，提升用户体验
- **结果缓存**：优化结果自动缓存，重复请求秒级返回
- **技能关键词提取**：自动识别简历中的技术栈和技能关键词，与职业测评联动推荐岗位

### 五、面经论坛社区

用户可发布和浏览面试经验分享帖，构建互助学习社区。

**功能亮点：**

- **发帖分享**：支持发布面经帖子，填写公司、岗位、面试时间、面试方式（远程/线下）、面试结果等详细信息
- **多维筛选**：支持按公司、岗位、年份、面试结果（Offer / 待定 / 拒绝 / 进行中）、面试方式、标签进行交叉组合筛选
- **排序方式**：支持最新排序和最热排序
- **互动功能**：点赞、评论（嵌套评论）、分享链接一键生成
- **标签系统**：自定义标签分类，热门标签统计展示
- **权限控制**：仅作者可删除自己的帖子和评论

### 六、管理后台

独立的管理员后台，提供系统级管理能力。

**功能亮点：**

- **数据仪表盘**：展示用户总数、帖子总数、面试场次、提交次数等关键运营指标
- **用户管理**：查看用户列表，支持激活/禁用账号操作
- **题目管理**：在线 CRUD 编程题目，支持设置难度、标签、测试用例、时间限制等
- **帖子管理**：审核和管理论坛帖子，支持删除违规内容

## 快速开始

### 环境要求

- **Node.js** ≥ 18
- **Python** ≥ 3.11
- **MySQL** ≥ 8.0
- **GCC / G++**（用于 C/C++ 代码判题，可选）
- **JDK** ≥ 11（用于 Java 代码判题，可选）

### 后端启动

```bash
# 进入后端目录
cd backEnd

# 创建虚拟环境
python -m venv .venv
.venv\Scripts\activate    # Windows
# source .venv/bin/activate  # Linux / macOS

# 安装依赖
pip install -r requirements.txt

# 创建 .env 配置文件（参考下方环境变量说明）

# 创建数据库
mysql -u root -p -e "CREATE DATABASE hr_interview CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 启动服务（自动建表 + 初始化种子数据）
uvicorn app.main:app --reload --port 8000
```

### 前端启动

```bash
# 进入前端目录
cd frontEnd

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端默认运行在 `http://localhost:5173`，通过 Vite 代理将 `/api` 请求转发到后端 `http://localhost:8000`。

### 环境变量配置

在后端根目录创建 `.env` 文件，配置以下必填项：

```env
# 数据库
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=hr_interview

# JWT 密钥（生产环境请替换为强随机字符串）
SECRET_KEY=your-secret-key

# DeepSeek API（简历分析、面试评分、AI 对话等功能需要）
DEEPSEEK_API_KEY=your-deepseek-api-key
DEEPSEEK_API_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash

# 讯飞语音听写（面试语音识别功能需要）
IFLYTEK_APP_ID=your-app-id
IFLYTEK_API_KEY=your-api-key
IFLYTEK_API_SECRET=your-api-secret
```

## 设计亮点

### 安全机制

- **密码安全**：使用 bcrypt 哈希算法存储密码，最大支持 72 字节安全截断
- **JWT 认证**：无状态 Token 认证，支持 24 小时过期自动刷新
- **代码执行沙箱**：多层安全拦截，每种语言独立配置危险操作黑名单（覆盖文件操作、进程注入、网络访问、系统命令、注册表操作等），临时目录隔离执行
- **面试防作弊**：全屏保护、切屏检测（5 次自动终止）、右键禁用、开发者工具拦截，违规操作统一 Toast 提示

### 性能优化

- **全异步架构**：FastAPI + SQLAlchemy 异步引擎 + aiomysql，高并发场景下无阻塞
- **SSE 流式输出**：AI 面试对话、简历优化、岗位推荐均采用 Server-Sent Events 流式传输，实时呈现生成过程
- **语音合成流式传输**：Edge-TTS 边生成边播放，无需等待完整音频文件
- **结果缓存**：简历优化和岗位推荐结果自动缓存到数据库，重复请求秒级响应
- **连接池管理**：SQLAlchemy 连接池（pool_size=10, max_overflow=20），数据库迁移自动预检（pool_pre_ping）

### 用户体验

- **字体大小调节**：面试选择题界面支持动态调整题目字号（A+ / A-），满足不同视觉需求
- **雷达图评分**：面试报告使用 ECharts 雷达图多维度可视化展示能力画像
- **MBTI 人格图标**：职业测评结果配合 16 种人格专属 SVG 图标展示
- **响应式设计**：Tailwind CSS 响应式类名，适配移动端、平板和桌面端
